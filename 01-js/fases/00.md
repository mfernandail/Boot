# Fase 0

## Execution Context

### Qué es el contexto de ejecución?

- Es el entorno donde se ejecuta tu código.
- Define qué variables y funciones están accesibles en un determinado punto del código.
- Cada vez que se llama a una función, se crea un nuevo contexto de ejecución.

## Call Stack

### ¿Qué es el Call Stack?

- Es una estructura de datos que sigue el orden de ejecución de las funciones.
- Funciona como una pila (LIFO - Last In, First Out) El último que entra, es el primero que sale.
- Cuando una función es llamada, se añade al tope del stack.
- Cuando una función termina su ejecución, se elimina del tope del stack.

## Hoisting

### ¿Qué es el Hoisting?

- Es un comportamiento de JavaScript donde las declaraciones de variables y funciones son "elevadas" al inicio de su contexto de ejecución.
- Las declaraciones de funciones son completamente elevadas, mientras que las declaraciones de variables solo elevan la declaración, no la inicialización.
- Esto significa que puedes usar funciones y variables antes de que sean declaradas en el código.
